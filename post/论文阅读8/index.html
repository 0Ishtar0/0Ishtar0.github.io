In-Place Test-Time Training

研究动机

LLM在预训练阶段把知识写入参数，推理阶段只读取上下文，模型权重本身不再变化。面对长上下文、持续变化、流式输入等场景时，模型只能依赖 attention 在上下文窗口内检索信息，而 attention 的复杂度和上下文窗口大小限制了模型处理超长、持续演化任务的能力。

Test-Time Training, TTT 提供了另一种思路：在推理阶段更新一小部分参数，即 fast weights，让模型把当前上下文中的信息临时写入参数中。但是，已有 TTT 方法直接用于当前 LLM 生态时有三个主要障碍：

  1. 架构不兼容：许多 TTT 方法引入特殊 recurrent / memory layer，甚至试图替换 attention，难以直接从已有 LLM checkpoint warm start。
  2. 计算效率低：经典 TTT 往往逐 token 更新 fast weights，顺序依赖强，不适合大规模并行。
  3. 目标函数不适合语言建模：传统 TTT 常用重构当前输入表示的目标，但自回归 LLM 的核心目标是预测下一个 token，两者不完全对齐。

本文目标：在不破坏现有 Transformer 主干的前提下，让 LLM 获得测试时动态适应能力。

背景介绍：TTT

Learning to (Learn at Test Time): RNNs with Expressive Hidden States 【ICML 2025】 此前应用于语言任务的 TTT 使用一组可快速更新的参数 \(W\)，也就是 fast weights。它们构成一个小型神经网络：

\[ f_W(\cdot): \mathbb{R}^d \rightarrow \mathbb{R}^d \]

对于输入序列：

\[ x = [x_1, x_2, \ldots, x_N] \]

每个 token 表示 \(x_i\) 会被投影成 query、key、value：

\[ q_i,\quad k_i,\quad v_i \]

TTT采用重建损失，用当前 token 的 key-value 对更新 fast weights：

\[ W_i \leftarrow W_{i-1} - \eta \nabla_W L\left(f_{W_{i-1}}(k_i), v_i\right) \]

其中\(L\)为自监督损失，常见选择是 MSE

直观理解：这一步把 \((k_i, v_i)\) 这个关联写进 fast weights。

更新后的 fast weights 被用来处理 query：

\[ o_i = f_{W_i}(q_i) \]

输出 \(o_i\) 会被送回主干网络，影响后续隐状态和最终 token 预测。

这种目标可以帮助模型存储上下文信息。

方法

Transformer 中常见的 gated MLP 可以写成：

\[ O = \phi(HW_{gate}^{\top}) \odot (HW_{up}^{\top}) W_{down}^{\top} \]

其中\(H \in \mathbb{R}^{n \times d_{model}}\)是输入的隐状态。

记 MLP 的中间激活定义为：

\[ Z = \phi(HW_{gate}^{\top}) \odot (HW_{up}^{\top}) \]

MLP 输出为：

\[ O = Z W_{down}^{\top} \]

文章将 \(W_{down}\)作为 fast weights，在推理时根据上下文动态更新。

设序列被划分为多个 chunk，每个 chunk 大小为 \(C\) （1024）。

对第 \(i\) 个 chunk，记：

\[ Z_{[i]},\quad V_{[i]},\quad O_{[i]} \]

分别是该 chunk 的中间激活、目标 value 和输出。

第 \(i\) 个 chunk 处理前的 fast weights 为：

\[ W_{down}^{(i)} \]

先用当前 fast weights 计算该 chunk 的 MLP 输出：

\[ O_{[i]} = Z_{[i]} \left(W_{down}^{(i)}\right)^{\top} \]

该输出进入 Transformer 主干，影响后续预测。

再用该 chunk 的 \(Z_{[i]}\) 和 \(V_{[i]}\) 更新 \(W_{down}\)：

\[ W_{down}^{(i+1)}= W_{down}^{(i)}- \eta \nabla_W L\left( Z_{[i]}\left(W_{down}^{(i)}\right)^{\top}, V_{[i]} \right) \]

其中$Z_{[i]} \rightarrow V_{[i]}$可以看做当前 chunk 写入 fast weights 的 key-value 关联。

本文认为传统重构目标不适合 LLM，因此设计了一个和 next-token prediction 对齐的目标。

目标 value 定义为：

\[ \hat V = \operatorname{Conv1D}(X_0) W_{target} \]

其中：

  • \(X_0 \in \mathbb{R}^{n \times d_{model}}\)：输入 token embedding；
  • \(\operatorname{Conv1D}(\cdot)\)：沿序列维度的一维卷积；
  • \(W_{target} \in \mathbb{R}^{d_{model} \times d_{model}}\)：可训练投影矩阵。

如果把 \(W_{target}\) 设置为恒等矩阵，并让 Conv1D 的卷积核只取下一个 token，那么：

\[ \hat v_t = E_{x_{t+1}} \]

这就是最简单的 next-token target。实际实现中，Conv1D 和 \(W_{target}\) 可以学习一个局部未来 token 的组合，跟随模型训练，属于slow weights。

本文采用负 Frobenius 内积作为损失，在这个损失下，chunk-wise 更新可简化为：

\[ W_{down}^{(i)}= W_{down}^{(i-1)}+ \eta \hat V_{[i]}^{\top} Z_{[i]} \]

理论分析

假设前文出现 key-value 对：

\[ (x_{t^*}, x_{t^*+1}) = (k^*, v^*) \]

后文查询位置 \(n\) 再次出现：

\[ x_n = k^* \]

模型应该预测：

\[ x_{n+1}=v^* \]

In-Place TTT 的 fast weight 更新为：

\[ \Delta W_{down}= \eta \sum_{t \in \text{prior chunks}} V_t Z_t^{\top} \]

该更新对查询位置 \(n\) 上 token \(w\) 的 logit 改变量为：

\[ \Delta l_n[w]= E_w^{\top} \Delta W_{down} Z_n \]

如果使用 LM-Aligned target：

\[ V_{t^*}^{LM}=E_{x_{t^*+1}}=E_{v^*} \]

那么关键更新项近似为：

\[ \Delta W_{down} \approx \eta E_{v^*} Z_{t^*}^{\top} \]

代入 logit 变化：

\[ \Delta l_n[w] \approx \eta (E_w^{\top}E_{v^*}) (Z_{t^*}^{\top}Z_n) \]

由于查询位置 \(x_n=k^*\) 与前文 key 位置 \(x_{t^*}=k^*\) 对齐，因此：

\[ Z_{t^*}^{\top}Z_n > 0 \]

对于正确 token \(w=v^*\)：

\[ E_{v^*}^{\top}E_{v^*} = \|E_{v^*}\|^2 \]

较大，因此正确 token 的 logit 会增加。

对于错误 token \(w \neq v^*\)，若不同 token embedding 近似正交，则：

\[ E_w^{\top}E_{v^*} \approx 0 \]

因此其他 token 的 logit 基本不变。

而传统重建损失写入的是：

\[ V_{t^*}^{rec}=E_{x_{t^*}}=E_{k^*} \]

它会增强 \(k^*\) 自己，而不是增强 \(v^*\)，所以不能保证提高正确下一个 token 的 logit。
